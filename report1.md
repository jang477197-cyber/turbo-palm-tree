# 피보나치 및 GCD 프로파일링 분석

## 2) 피보나치 복잡도 및 GCD 분석

### ● 실험 방법

피보나치 수열의 n번째 값을 F(n)이라 할 때,
n을 5부터 순서대로 증가시키며 GCD(F(n), F(n-1))을 계산하였다.

또한 재귀적 피보나치 함수는 n이 증가할수록 실행 시간이 급격히 증가하므로,
프로그램의 실행 시간을 고려하여 n의 범위를 5부터 30까지로 제한하였다.

피보나치 수열은 재귀적 방법으로 구현하였다.

또한 Visual Studio 프로파일러를 사용할 수 없는 환경이므로,
time.h 라이브러리의 clock() 함수를 이용하여 프로그램의 시작과 끝의 시간 차이를 측정하고,
이를 출력하여 프로파일링 그래프 이미지를 대신하였다.

---

### ● 사용 코드

```c
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

```c
int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);
}
```

### ● CPU 사용량 프로파일링

clock() 함수를 이용하여 실행 시간을 측정하였다.
실행 결과, n이 증가할수록 실행 시간이 점점 증가하는 것을 확인할 수 있다.

이는 재귀적으로 구현된 피보나치 함수에서 동일한 계산이 반복되기 때문이다.

---

### ● 메모리 사용량 프로파일링

재귀 호출이 증가함에 따라 함수 호출 스택이 계속 쌓이게 된다.
따라서 입력값 n이 증가할수록 메모리 사용량도 증가한다.

재귀 깊이는 n에 비례하므로 메모리 사용량은 O(n)이다.

---

### ● 시간복잡도 검증

재귀 피보나치 함수는 중복 호출이 많기 때문에 실행 시간이 급격히 증가하며,
실험 결과에서도 n이 증가할수록 실행 시간이 빠르게 증가하는 것을 확인할 수 있다.

따라서 피보나치 수열의 시간복잡도는 다음과 같다.

→ **O(2ⁿ)**

GCD 알고리즘은 유클리드 호제법을 사용하며,
반복 횟수가 빠르게 감소하므로 다음과 같은 시간복잡도를 가진다.

→ **O(log n)**

전체 수행 시간은 피보나치 함수가 지배적이므로

→ **O(2ⁿ)**

---

### ● 결론

재귀적으로 구현된 피보나치 수열은 매우 비효율적인 알고리즘으로,
입력값이 증가할수록 CPU 사용량과 실행 시간이 급격히 증가한다.

또한 재귀 호출로 인해 메모리 사용량도 증가한다.

실험 결과와 분석을 통해 피보나치 수열의 시간복잡도는 O(2ⁿ),
GCD 알고리즘의 시간복잡도는 O(log n)임을 확인할 수 있었다.
