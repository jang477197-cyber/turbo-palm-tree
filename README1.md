# 📌 피보나치 수열 반복문 vs 재귀 비교

## 1. 과제 개요
본 과제에서는 피보나치 수열을 반복문 방식과 재귀 방식으로 각각 구현하고,  
time.h 라이브러리의 clock() 함수를 이용하여 프로그램의 실행 시간을 측정하였다.  
측정된 실행 시간 데이터를 바탕으로 두 방식의 성능을 분석하고 비교하였다.

---

## 2. 피보나치 수열 정의

- F(1) = 1  
- F(2) = 1  
- F(n) = F(n-1) + F(n-2)

---

## 3. 실행 결과 (표)

| N | 반복문 방식 실행 시간 (초) | 재귀 방식 실행 시간 (초) |
|---|--------------------------|--------------------------|
| 10 | 0.000000 | 0.000000 |
| 20 | 0.000000 | 0.001000 |
| 30 | 0.000000 | 0.022000 |

---

## 4. 반복문 방식 분석

반복문 방식은 이전에 계산된 두 개의 값을 이용하여 다음 값을 계산하는 방식이다.  
한 번의 반복마다 하나의 값만 계산되며, 이미 계산된 값을 다시 사용하는 구조이기 때문에  
불필요한 연산이 발생하지 않는다.  

이 방식의 시간 복잡도는 O(n)으로, 입력값이 증가함에 따라 실행 시간이 선형적으로 증가한다.  
따라서 매우 효율적인 알고리즘이라 할 수 있다.

---

## 5. 재귀 방식 분석

재귀 방식은 함수가 자기 자신을 호출하여 값을 계산하는 방식이다.  
이 과정에서 동일한 값이 여러 번 반복 계산되는 문제가 발생한다.  

예를 들어 F(5)를 계산할 경우, F(3)과 F(2)가 여러 번 호출되며  
이로 인해 불필요한 연산이 크게 증가한다.  

이 방식의 시간 복잡도는 O(2^n)으로, 입력값이 증가할수록 실행 시간이 급격히 증가한다.

---

## 6. 두 방식의 비교

반복문 방식과 재귀 방식을 비교하면 다음과 같은 차이점이 있다.

- 반복문 방식은 중복 계산이 발생하지 않으며 효율적이다.
- 재귀 방식은 동일한 계산을 반복 수행하여 비효율적이다.
- 입력값이 작을 때는 두 방식의 차이가 크지 않지만,
  입력값이 커질수록 실행 시간의 차이가 매우 크게 나타난다.
- 실제 측정 결과에서도 재귀 방식이 반복문 방식보다 훨씬 긴 시간이 소요되었다.

---

## 7. 결론

clock() 함수를 이용하여 실행 시간을 측정한 결과,  
반복문 방식이 재귀 방식보다 훨씬 빠르고 효율적인 것을 확인할 수 있었다.  

따라서 피보나치 수열과 같은 문제를 해결할 때는  
반복문 방식을 사용하는 것이 더 적합하다.

※ 실행 시간은 컴퓨터 환경에 따라 다소 차이가 있을 수 있다.
